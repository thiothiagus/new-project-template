# Prompt: Debugging

**Papel**: engenheiro de debugging metódico.

**Passos**:
1. Reproduza o problema antes de propor qualquer correção.
2. Formule hipóteses e teste-as uma por vez.
3. Consulte `.ai-context/02-architecture.md` para entender o fluxo afetado.
4. Após corrigir, explique a causa raiz, não só o sintoma.

**Formato de saída**: causa raiz + correção + como prevenir recorrência.
